# Preliminary Findings — Overlap-Add Streaming Filtering

Generated by `run_boundary_experiment.py` (synthetic ground truth, real `domain` production
filters). Metrics in `results/metrics.csv`; figures in `results/`.

## Setup
- 120 s, 4-channel synthetic EEG, 200 Hz, with band oscillations, 1/f noise, and 50 Hz mains.
- Filters: the production `domain.helpers.filters.create_bandpass_filter` (FIR order 200,
  0.5–30 Hz) + `create_notch_filter` (50 Hz). Ground truth = whole-signal filtering.
- Strategies: naive per-window vs overlap-add (extended window, keep center) — the production
  fix from `file-service/changelogs/OVERLAP_ADD_FIX_APPLIED.md`.

## Result 1 — Overlap-add eliminates seam artifacts (zero-phase regime, 8 s windows)
- Boundary RMSE vs ground truth: **naive 14.93 µV → overlap-add (0.5 s) 0.275 µV = 34.7 dB
  reduction**; overlap-add (1.0 s) → 3.3e-4 µV (≈ eliminated).
- Latency cost is **~1%** (0.556 → 0.564 ms/window): the improvement is effectively free.
- Naive filtering concentrates error at seams (boundary-to-interior ratio +60 dB);
  overlap-add removes it (`fig_seam.png`, `fig_profile.png`, `fig_summary.png`).

## Result 2 — The `filtfilt` length floor (a regime effect, conditional production concern)
- `create_bandpass_filter` uses zero-phase `filtfilt` only when the input has
  **≥ 3 × len(taps) = 606 samples (3.03 s @ 200 Hz)**; shorter inputs **silently** fall back to
  a phase-distorting causal `lfilter` path (`domain/domain/helpers/filters.py:229-238`).
- The streaming path filters a window of `db_chunk_size + 2 × overlap`. The **default**
  `db_chunk_size = 10 s` (`file-service/src/routes/data_streaming.py:367-370`) with
  `_FILTER_OVERLAP_S = 0.5 s` (`.../streaming/helpers.py:69`) → 11 s = 2200 samples → **filtfilt
  runs; the default is zero-phase and correct.**
- **The concern is the configurable small-chunk regime.** `chunk_size` is clamped only to
  `[0.5, 60] s`, so a client requesting a small chunk silently enters the fallback regime:
  chunk = 1 s + overlap 0.5 s = **400 samples (< 606)** → `filtfilt_active = 0`, interior RMSE
  30.7 µV (distorted everywhere, not just seams). To stay zero-phase at a 1 s chunk, overlap
  must be **≥ 1.01 s each side**, or keep `chunk + 2 × overlap ≥ 3.03 s`. The `metrics.csv`
  `filtfilt_active` column marks the regime boundary.

## So what (paper framing)
The contribution is not merely "overlap-add fixes seams." It is a characterization of the
interaction between **FIR order**, **`filtfilt`'s 3×(order+1) length floor**, **display window
size**, and **overlap**, which jointly determine (a) whether streaming EEG filtering is
zero-phase at all and (b) whether seam transients are suppressed. We quantify both regimes and
derive the minimum context, with a concrete recommendation for production parameters.

## Product note (for the engineering backlog)
With the current defaults (order-200 FIR, ~1 s windows, 0.5 s overlap), streaming filtering
does not reach `filtfilt` and is therefore not zero-phase. Options: increase overlap to
≥ 1.01 s/side at 1 s windows, increase the display window, or lower the FIR order.

## Result 3 — Filter magnitude fidelity and event mis-timing (`exp_filter_fidelity.py`)
- **Magnitude (zero-phase path):** passband (1–25 Hz) gain −0.64 ± 1.54 dB (ideal 0 dB),
  50 Hz notch −120 dB, 60 Hz stopband −120 dB → the intended response is realized
  (`fig_magnitude.png`).
- **Temporal shift of a 20 ms transient (proxy for an epileptiform spike):**
  - zero-phase (`filtfilt`) path: **0 ms** (no shift).
  - short-window fallback (`lfilter`) path: **+500 ms** — the linear-phase FIR group delay
    (~(taps−1)/2 ≈ 502 ms) is **not compensated** (`fig_transient_shift.png`).
- Combined with Result 2: whenever the streaming filter enters the fallback regime (small
  `chunk_size`), streamed/filtered EEG is **temporally shifted by ~0.5 s** — events appear half
  a second from their true time. At the default 10 s chunk this does not occur.

## Result 4 — Channel canonicalization and feature integrity (`exp_channels_features.py`)
- **Canonicalization:** family-classification accuracy **89.5%** on 19 real-world channel-name
  variants. Of the two disagreements: `EEG A1-REF` → `ref_site` is actually correct (ear/mastoid
  reference, not scalp EEG); `POL Fp1` → `aux` is a **genuine gap** — the Nihon-Kohden `POL`
  prefix is not stripped, so such EEG channels are misclassified and would be wrongly excluded
  from EEG montages.
- **Feature integrity:** computing an average-reference montage **without** excluding a
  mislabeled ECG channel inflates C3 alpha bandpower by **+48.7%** (58.68 vs 39.45). Correct
  quantitative features require channel-aware classification before montage.
