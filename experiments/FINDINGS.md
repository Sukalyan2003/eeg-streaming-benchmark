# Findings — Windowed EEG Filtering and Spectral Estimation

Generated by the scripts in this directory. Every experiment depends only on NumPy, SciPy, and
(for real-data loading) MNE — the reference DSP primitives live in `dsp.py`, the runtime guardrail
in `safe_window.py`, and the EDF-duration benchmark in `benchmark_edf_lengths.py`. Metrics and
figures are written to `results/`.

Environment: Python 3.12, numpy 2.4.1, scipy 1.17.0, mne 1.11.0.

## Setup
- 120 s, 4-channel synthetic EEG, 200 Hz, with band oscillations, 1/f noise, and 50 Hz mains
  (`synth.py`).
- Reference filters (`dsp.py`): FIR bandpass (order 200 -> 202 taps, Hamming `firwin`,
  0.5–30 Hz) applied with zero-phase `filtfilt`, and an IIR notch (`iirnotch`, 50 Hz). The
  bandpass uses `filtfilt` when the window is long enough and falls back to a causal `lfilter`
  when it is not longer than `3 x len(taps)` (the documented `filtfilt` padding requirement).
- Windowing strategies (`windowing.py`): whole-signal (zero-phase reference), naive per-window,
  and overlap-add (filter an extended window, keep the center).

## Result 1 — Overlap-add eliminates seam artifacts (zero-phase regime, 8 s windows)
- Boundary RMSE vs ground truth: **naive 14.93 µV → overlap-add (0.5 s) 0.275 µV = 34.7 dB
  reduction**; overlap-add (1.0 s) → ≈ 0 (eliminated).
- Latency cost is **a few percent** (0.553 → 0.563 ms/window in the latest run): the improvement
  is effectively free.
- Naive filtering concentrates error at seams; overlap-add removes it (`fig_seam.png`,
  `fig_profile.png`, `fig_summary.png`).

## Result 2 — The zero-phase length floor and a design criterion
- `filtfilt` requires the window to exceed its default padding length, `3 x len(taps) = 606
  samples` for an order-200 FIR; the shortest valid zero-phase input is therefore **607 samples
  (3.04 s @ 200 Hz)**. A shorter window cannot be zero-phase, so a streaming filter that must
  return a window has to use a causal filter instead.
- A windowed filter processes a span of `chunk + 2 x overlap`. Large windows clear the floor and
  are zero-phase; a small window (e.g. 1 s chunk + 0.5 s overlap = 400 samples < 607) is below
  it, and on the causal path the error is large **throughout** the window (interior RMSE
  30.7 µV), not just at seams.
- **Design criterion:** keep `(chunk + 2 x overlap) x fs > 3 x len(taps)` to stay zero-phase
  (>= 1.02 s overlap each side at a 1 s chunk). The `metrics.csv` `filtfilt_active` column marks
  the regime boundary.

## Result 3 — Filter magnitude fidelity and event mis-timing (`exp_filter_fidelity.py`)
- **Magnitude (zero-phase path):** passband (1–25 Hz) gain −0.64 ± 1.54 dB, 50 Hz notch −120 dB,
  60 Hz stopband −120 dB → the intended response is realized (`fig_magnitude.png`).
- **Temporal shift of a 20 ms transient (proxy for an epileptiform spike):**
  - zero-phase (`filtfilt`) path: **0 ms** (no shift).
  - causal-fallback (`lfilter`) path: **+500 ms** — the linear-phase FIR group delay
    `(taps−1)/2 ≈ 502 ms` is **not compensated** (`fig_transient_shift.png`).
- Combined with Result 2: whenever a window is short enough to force the causal path, displayed
  events are mis-timed by ~0.5 s; on a zero-phase window the timing is exact.

## Result 4 — Generalization across FIR order, the transition, seed robustness (`exp_regime.py`)
- **FIR-order sweep (8 orders, 50–500):** the zero-phase pad length is exactly `3 x taps`, the
  shortest valid input is `3 x taps + 1`, and the fallback shift is `(taps−1)/2/fs`, all scaling
  with order: order 50 → padlen 156 samples, shift 130 ms; 100 → 306, 255 ms; 200 → 606, 505 ms;
  300 → 906, 755 ms; 400 → 1206, 1005 ms; 500 → 1506, 1255 ms. This generalizes the criterion
  **`chunk + 2 x overlap > 3 x taps`** to any FIR order (`regime_order_sweep.csv`,
  `fig_order_floor.png`).
- **Transition:** sweeping the total filtered length through the floor shows boundary error
  collapse sharply at `L = 3 x taps` (`regime_transition.csv`, `fig_transition.png`).
- **Seed robustness (24 seeds, 8 s windows):** naive boundary RMSE **14.4 ± 3.9 µV**, overlap-add
  **0.271 ± 0.032 µV**, reduction **34.3 ± 2.4 dB** — the headline effect is stable.

## Result 5 — Cascaded-tapering pitfall in PSD estimation (`exp_bandpower_equiv.py`)
- Applying a Hann window to the whole analysis window **and then** an estimator that already
  tapers (`scipy.welch`'s per-segment Hann; a multitaper estimator's DPSS tapers) tapers the
  signal twice and biases relative band power.
- **Welch:** a doubly-tapered estimate differs from MNE `psd_array_welch` by up to **11.9
  percentage points** (on the dominant band); tapering once (welch only) matches MNE to **0.64
  pp** (Pearson r = 0.99997). See `bandpower_equiv.csv`.
- **Multitaper:** pre-windowing before the DPSS tapers biases relative band power by up to
  **9.08 pp**; without the pre-window it matches the plain multitaper reference (0.0 pp). The
  bias is therefore a property of cascaded tapering, not of one estimator.
- **Rule:** taper exactly once.

## Result 6 — Multi-subject, two-dataset real-data confirmation
External validity across two open datasets with different populations, montages, and sampling
rates (raw EDFs gitignored under `data/`).

**CHB-MIT** (pediatric epilepsy, bipolar, 256 Hz; first recording of each subject; `exp_multisubject.py`):
- **R1 confirmed:** overlap-add reduces window-seam boundary RMSE by **50.6 ± 2.2 dB** (mean ± SD
  across **10** subjects) — comparable to or larger than synthetic, with near-zero interior error.
- **R3 confirmed:** the causal-fallback group-delay shift, by cross-correlation against the
  zero-phase output, is **389 ± 7 ms**, matching `(taps−1)/2/fs` (|measured − theory| =
  **4.0 ± 6.5 ms**). The pad length (606 samples) is **2.37 s** at 256 Hz; the shortest valid
  zero-phase input is 607 samples.
- Per-subject values: `multisubject_chbmit.csv`; figure `fig_realdata_seam.png` (`exp_realdata.py`).

**Sleep-EDF Expanded** (sleep, Fpz-Cz/Pz-Oz, 100 Hz; `exp_sleepedf.py`):
- **R1 confirmed:** overlap-add reduction **23.4 ± 2.0 dB** (mean ± SD across **8** subjects).
- **R3 confirmed:** fallback shift **1001 ± 2 ms** vs theory `(taps−1)/2/fs` (|measured − theory| =
  **3.8 ± 2.2 ms**). The same 606-sample pad length is **6.06 s** at 100 Hz, and the shortest
  valid input is 607 samples — making the `fs`-dependence explicit (longer floor and ~1 s
  mis-timing at the lower rate).
- Per-subject values: `sleepedf_subjects.csv`.

## Result 7 — Safe-window scheduler and EDF-duration runtime benchmark
`safe_window.py` turns the zero-phase criterion into a runtime plan:

- For a 1 s chunk and requested 0.5 s overlap with an order-200 FIR:
  - 200 Hz: requested span = 400 samples (unsafe), scheduled overlap = **1.02 s** per side.
  - 256 Hz: requested span = 512 samples (unsafe), scheduled overlap = **0.6875 s** per side.
  - 100 Hz: requested span = 200 samples (unsafe), scheduled overlap = **2.54 s** per side.
- It also exposes first/last-window boundary status; with real-sample context only, record edges
  still need an explicit boundary policy or extra one-sided context.

`benchmark_edf_lengths.py` adds end-to-end timing evidence over **all 18 local recordings** —
**10 CHB-MIT** (256 Hz, 8 channels) and **8 Sleep-EDF** (100 Hz, 2 channels) — sweeping segment
duration (30–600 s), FIR order (100–400), and channel count (1–16), reported as mean ± SD across
the files of each dataset (outputs `edf_length_benchmark.csv`, `edf_order_sweep.csv`,
`edf_channel_sweep.csv`, three figures, and `edf_length_benchmark_summary.txt`).

Timing runs **without** memory instrumentation — `tracemalloc` in the timed region inflated wall
time ~2.5×, so memory is reported from array `nbytes` instead.

- **Throughput (600 s segments, mean ± SD across files):**
  - CHB-MIT: scheduled-overlap **3.05 ± 0.06 s** (RTF 5.1×10⁻³, ~18 s/EEG-hour, interior
    zero-phase safe); fixed-overlap **1.25 ± 0.02 s** (~2.4× faster but unsafe, causal-fallback
    regime); whole-signal **0.79 ± 0.01 s** (offline reference). Peak array memory ≈ 20 MB.
  - Sleep-EDF: scheduled-overlap **0.74 ± 0.01 s** (RTF 1.2×10⁻³); whole-signal 0.17 s.
  - Every streaming path is 2–3 orders of magnitude faster than real time.
- **Scheduler is fs-aware:** for a 1 s chunk + 0.5 s requested context the configured span is
  below the 607-sample floor at both rates, so context is raised to **0.6875 s** per side at
  256 Hz but **2.54 s** at 100 Hz — the runtime face of the R6 fs-dependence.
- **Order scaling (120 s):** scheduled cost rises with order because the scheduler grows context
  to clear the larger floor — CHB-MIT 0.37 s (order 100) → 1.89 s (order 400); whole-signal stays
  flat (~0.17 s) since it filters once.
- **Channel scaling (120 s):** processing time is linear in channel count for every method
  (CHB-MIT scheduled: 0.08 s at 1 ch → 1.22 s at 16 ch).
- Single-machine benchmark; method ordering and linear scaling are machine-independent, absolute
  times are not.

## Result 8 — Directly measured per-window latency (`benchmark_latency.py`)
Throughput is not interactivity. This benchmark filters **one display window at a time**, timing
each all-channel request independently, and reports the distribution against a 60 Hz display
refresh (16.7 ms). Over all 18 files (outputs `latency_benchmark.csv`, `fig_latency_benchmark.png`):
- CHB-MIT (256 Hz, 8 ch), scheduled-overlap: **4.2 ms** median to filter a 1 s window, **5.7 ms**
  including per-window Welch band power (worst-case p95 **6.4 ms**).
- Sleep-EDF (100 Hz, 2 ch), scheduled-overlap: **1.1 ms** filter / **1.5 ms** with features.
- **Every method on both datasets stays within one 60 Hz refresh.** The directly measured latency
  agrees with R7's amortized estimate, confirming the throughput figure hid no per-window cost.

## Result 9 — Record-edge boundary policies (`exp_edge_policy.py`)
The scheduler's guarantee is interior-only; the first/last window of a record has one-sided
context. We measure three policies for the edge window vs the whole-signal zero-phase reference
(boundary RMSE in µV; outputs `edge_policy.csv`, `fig_edge_policy.png`):
- **Interior-of-record view edge** (real neighbours exist): **prefetch** is essentially exact —
  CHB-MIT **0.001 µV** vs **56 µV** causal (**93.6 dB**), Sleep-EDF 0.001 vs 22 µV (**88.6 dB**);
  reflection-zero-phase gives ~23 dB.
- **True record boundary** (no neighbours): reflection zero-phase avoids the causal fallback and
  reduces edge error by **7.2 dB** (CHB-MIT, residual **17.9 ± 58.4 µV**) / **16.2 dB** (Sleep-EDF,
  residual **4.0 ± 4.5 µV**), but the residual remains because reflection only approximates the
  absent samples — it is not exact like prefetch.
- **Rule:** prefetch real context when it exists (interior edges become exact), reflect-pad at true
  record boundaries to avoid the causal regime, never silently take the causal path. The scheduled
  configuration is then operationally zero-phase at interior windows, with reduced (not eliminated)
  error at the two true record boundaries.

## Exploratory (not featured in the paper) — montage hygiene
`exp_channels_features.py` / `exp_realdata_features.py` show that including a non-EEG channel in
an average reference injects a scaled copy of it (weight `1/(N+1)`) into every re-referenced EEG
channel; synthetic leakage correlation tracks `~2/(N+1)` (0.50 at N=3 → 0.07 at N=31). On real
recordings the effect is confounded because scalp EEG already carries cardiac artifact (an
EEG-only reference is itself ~0.46-correlated with the EKG), so the magnitude is montage- and
recording-dependent. The mechanism is robust but the quantitative result is not clean enough to
feature; kept here for completeness.

## Reproduce
Exact package versions are pinned in `requirements.txt`. On some platforms MNE's `numba`
backend fails to import; if so, export `NUMBA_DISABLE_JIT=1` before running (the benchmark sets
this itself). The real-data cohort is fixed by `data/MANIFEST.csv` (10 CHB-MIT + 8 Sleep-EDF
files with SHA-256 checksums); fetch exactly those files so the multi-subject and benchmark
cohorts match the paper rather than "whatever is in `data/`".
```bash
cd Research/Option-2-Non-AI-Based/experiments
export NUMBA_DISABLE_JIT=1            # only if MNE import fails on your platform
python3 run_boundary_experiment.py   # Results 1–2 (overlap-add + zero-phase floor)
python3 exp_filter_fidelity.py       # Result 3 (magnitude + event timing)
python3 exp_regime.py                # Result 4 (FIR-order sweep, transition, seed robustness)
python3 exp_bandpower_equiv.py       # Result 5 (cascaded-tapering pitfall; welch + multitaper)
python3 exp_multisubject.py          # Result 6a (multi-subject CHB-MIT; downloads EDFs to data/)
python3 exp_sleepedf.py              # Result 6b (second dataset: Sleep-EDF, 100 Hz)
python3 exp_realdata.py              # representative CHB-MIT seam figure (needs chb01_01.edf)
python3 safe_window.py --fs 256 --chunk-s 1 --overlap-s 0.5
python3 benchmark_edf_lengths.py   # Result 7 (multi-file EDF runtime: duration/order/channel sweeps)
python3 benchmark_latency.py       # Result 8 (directly measured per-window latency)
python3 exp_edge_policy.py         # Result 9 (record-edge boundary policies)
```
